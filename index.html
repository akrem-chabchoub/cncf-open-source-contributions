Chapter 1

What is Open Source?

📖

Open Source Software

Source code that anyone can inspect, modify, and distribute. Governed by licenses like Apache 2.0, MIT, or GPL.

☸️

CNCF

Cloud Native Computing Foundation — home to Kubernetes, Prometheus, Envoy, Argo, Kubeflow, and 200+ more projects.

🤖

Kubeflow

The ML toolkit for Kubernetes. Pipelines, training operators, model serving — the full ML lifecycle on K8s.

🌍

Why it matters

Runs the internet's most critical infrastructure. Contributing means your code ships to millions of users.

Chapter 3

Finding Good Issues

🏷️

Labels to hunt

  • good first issue
  • help wanted
  • documentation
  • easy
🔍

Where to look

GitHub issue tracker, project roadmap, OWNERS files. Also: tests, linting, CI fixes — unglamorous but valued.

🎯

What makes it good

  • Scoped to one file/package
  • Has a clear acceptance criterion
  • Maintainer responded recently
  • Relevant to skills you have

Chapter 4

Working with Maintainers

Do's ✅

  • Comment on the issue before opening a PR
  • Ask one clear question at a time
  • Reference the issue number in your PR
  • Respond to review comments quickly
  • Be patient — maintainers are volunteers

Don'ts ❌

  • ✗ Open a PR with "big refactor" without discussion
  • ✗ Ping maintainers daily asking for merge
  • ✗ Take feedback personally
  • ✗ Ignore CI failures on your PR
"Maintainers don't owe you a review. They choose to give you one. Respect that."
